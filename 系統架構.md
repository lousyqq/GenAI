# GenAI 系統架構

> 模組清單與資料流向的精簡總覽。開發規範見 `CLAUDE.md`；DB schema 見 `DB_Table.md`。

## 技術棧
- **後端**：ASP.NET Core (.NET 9.0)、EF Core（無 Migrations，SchemaBootstrap 冪等補齊）、Serilog、Swagger
- **資料庫**：MSSQL `Sariel/GenAI`（19 張表）；人事資料唯讀來源 `[WEB].[dbo].[notes_person]`
- **前端**：純靜態原生 JS ES Modules（無框架）+ Bootstrap 5 + DataTables + FontAwesome；`?v=` 版本號 cache-busting
- **認證**：Cookie（主）+ Windows Negotiate（僅 `/api/Auth/WhoAmI` 觸發）

## 關鍵資料流
1. **登入流**：瀏覽器 → `WhoAmI`（401→Negotiate 挑戰）→ 剝網域前綴 → `FindAccountAsync`（無帳號→自動開戶+`role_1`；notes_person 補姓名部門）→ `AccountOverrides` 覆寫 → 發 Cookie → 前端 `completeLoginAfterAuth`（刷新 CSRF＋補抓 InitialData）。
2. **每請求權限同步**：Cookie → `OnValidatePrincipal` → 比對 `Accounts`＋`AccountOverrides` → 角色異動即 `ReplacePrincipal` 續發、帳號失效即 RejectPrincipal。
3. **初始資料流**：`main.js` → `MyProfile` 探測（未登入直接走登入流，避免無效 401）→ `/Settings/GetInitialData`（SettingsService 以 ETag 快取；任何權限寫入經 `CacheInvalidationInterceptor`/`Invalidate*` 換 ETag）→ `appState` → 各 render 模組。
4. **選單可見/委派寫入**：讀 = `GetVisibleMenuIdsAsync`（(ETag, empId) 快取）；寫 = `MenusController` → `MenuService` →（先全部檢查後動 DB）`MenuAuthService`：admin 全可、自建可改、委派目錄子樹可管（動他人內容需 `CanEditOthers`）、無父項新增需為委派身分；非 admin 的 ACL 欄位後端強制清空。
5. **使用率統計流**：前端登入後 `pingSiteVisitor` → `Stats/Ping`（身分只信 Claim；匿名記 ANONYMOUS）→ upsert `SiteVisitorDailyStats`（主鍵 StatDate+EmpId，併發衝突優雅重試）→ admin 於統計頁讀 Summary/Daily（全量明細）/Monthly（SQL 端分組）/Export（CsvField 安全編碼）。
6. **稽核流**：每請求 → `ActivityLoggingMiddleware` → `ActivityLogQueue`（Channel）→ `ActivityLogProcessor`（HostedService 批次落盤 `UserActivityLogs`）→ `ActivityLogPurgeService` 依 `ActivityLog:RetentionDays` 每日清理。

## 安全機制
- **CSRF**：`X-Requested-With` + antiforgery token（登入後刷新）；驗證位於 Auth 之後的中介層。
- **Rate Limit**：`/api/Auth/Login` 每 IP 60 秒 10 次（429 + Retry-After）。
- **Production Guard**：非 Development 啟動時驗證高風險設定（TestAccounts/EmergencyAdmin/LDAP placeholder/弱密碼連線字串），違規拒絕啟動。
- **授權基線**：全部 Controller 至少 `[Authorize]`（Stats 查詢/匯出鎖 admin；僅 WhoAmI/Login/Ping 等少數端點可匿名）。

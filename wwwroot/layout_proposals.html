UMC GenAI 投影機版面重構提案

目標：在大螢幕投影環境下呈現專業、高對比的入口框架，並將最大空間讓給嵌入式看板內容 — 框架是配角，內容才是主角。以下 3 個範本由「改動最小」到「最沉浸」排列，可依簡報場景混用（範本 A 為日常操作、範本 C 為投影展示）。

投影機高對比（實色、無半透明文字） 內容占比 85% → 100% 框架極簡、不喧賓奪主 保留現有選單/權限機制

範本 A｜精簡單列導覽列

把現有雙列 72px 導覽列合併為一列 44px，主選單頁籤直接放在 Logo 旁；側欄保留但可一鍵收合。改動成本最低、日常操作體驗不變。
  • 導覽列 72px → 44px：Logo、主選單頁籤、麵包屑、使用者資訊全部單列排開。
  • 選單文字改實色白 #FFFFFF + 粗體，active 用實色底 #1D3A5F + 3px 亮青色底線，投影機上輪廓清晰。
  • 側欄可一鍵收合（現有 toggleSidebar 已具備）：收合後內容直接吃滿整寬。
  • iframe 頁隱藏 iframe-header：頁面標題已在導覽列麵包屑呈現，不重複占 33px。
  • 意見箱、系統設定收進右側工具區，不佔獨立列。

範本 B｜圖示側欄 Icon Rail

仿 Azure Portal / Grafana 的企業級版型：左側 56px 深色圖示欄常駐，滑鼠懸停展開飛出選單；上方僅留 40px 淺色狀態列。左右視覺重心穩定、看起來最「產品化」。
  • 側欄 200px → 56px 圖示欄：16 個 folder（DF1~TF6）以縮寫代號呈現，hover 飛出完整子選單（overlay，不推擠內容）。
  • 上方 40px 淺色列只放：目前頁面標題 + 路徑 + 使用者。深色圖示欄與淺色頂列形成 L 型框，專業感強。
  • 當前位置雙重標示：圖示欄亮藍底 + 頂列標題，主管掃一眼即知在哪個廠區模組。
  • 適合子選單數量多、但同時只看一個看板的使用情境。

範本 C｜沉浸簡報模式（投影專用）

內容 100% 滿版，僅保留一枚頂部浮動膠囊（Logo + 頁名 + 時鐘 + 連線狀態燈），數秒後自動淡出；滑鼠移到螢幕上緣喚出完整選單。給高階主管看的就是這一幕。
  • 零常駐框架：navbar、sidebar 全部隱藏，靠現有 edge-trigger（上緣/左緣感應）喚出。
  • 浮動膠囊列：綠色連線燈讓主管確認「這是即時資料」；時鐘方便會議對時；閒置 5 秒自動淡出。
  • 建議做成一鍵切換：在範本 A/B 的工具區放「投影模式」按鈕（現有 FOCUS 全螢幕機制可直接升級），會議開始前按一下即可。
  • 可加輪播：多看板每 N 分鐘自動切換，適合戰情室電視牆。

建議實施路徑

第一階段（1 天內）：套用「配色準則」— 只改 CSS（navbar.css / sidebar.css / variables.css），半透明文字全面改實色、字級加大。不動任何版面結構，立刻解決投影機看不清的問題。

第二階段：範本 A — 導覽列併為單列 44px + iframe 頁去掉重複標題列，內容占比 85.6%（收合側欄 95.6%）。HTML 結構變動小，現有 JS（toggleSidebar / selectTopMenu / 麵包屑）都能沿用。

第三階段：範本 C 投影模式 — 把現有 FOCUS 全螢幕升級成「浮動膠囊 + 自動淡出」的簡報模式，會議投影一鍵切換、內容 100%。範本 B 則視主管回饋決定是否投資（最有產品感，但 sidebar 需重寫）。
